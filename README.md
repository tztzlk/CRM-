# CRM-система для малого бизнеса

Дипломный проект. Веб-приложение для учёта клиентов, управления сделками и анализа продаж.

Стек: **Django 5 · PostgreSQL/SQLite · Bootstrap 5 · Chart.js · SortableJS**.

## Возможности

- 4.1.1 **Клиенты** — добавление, редактирование, удаление, поиск, фильтр по источнику, карточка клиента
- 4.1.2 **Сделки** — CRUD, статусы (открыта / успех / отказ), привязка к клиенту и менеджеру
- 4.1.3 **Воронка продаж** — канбан-доска с **drag-and-drop** между этапами; настраиваемые этапы
- 4.1.4 **История взаимодействий** — звонки, email, встречи, заметки, сообщения с лентой событий
- 4.1.5 **Пользователи и роли** — три роли: администратор / менеджер / наблюдатель
- 4.1.6 **Отчёты** — конверсия, распределение по этапам, выручка по месяцам, ТОП-5 менеджеров, экспорт в CSV
- Сводная панель (Dashboard) с ключевыми метриками

## Быстрый старт (Windows / PowerShell)

```powershell
# 1. Клонировать / скопировать проект
cd C:\Users\User\CRM

# 2. (рекомендуется) создать venv
python -m venv venv
.\venv\Scripts\Activate.ps1

# 3. Установить зависимости
pip install -r requirements.txt

# 4. (опционально) скопировать .env
Copy-Item .env.example .env

# 5. Применить миграции
python manage.py migrate

# 6. Загрузить демо-данные
python manage.py seed_demo

# 7. Запустить сервер
python manage.py runserver
```

Открыть [http://127.0.0.1:8000](http://127.0.0.1:8000).

## Демо-аккаунты

| Логин      | Пароль       | Роль          | Что доступно                             |
|------------|--------------|---------------|------------------------------------------|
| `admin`    | `admin123`   | Администратор | Всё, включая пользователей и этапы       |
| `manager`  | `manager123` | Менеджер      | CRUD клиентов, сделок, истории           |
| `manager2` | `manager123` | Менеджер      | То же, для демонстрации второго менеджера|
| `viewer`   | `viewer123`  | Наблюдатель   | Только просмотр и отчёты                 |

> Команда `python manage.py seed_demo --reset` пересоздаёт данные.

## Использование PostgreSQL

В файле `.env`:

```
DATABASE_URL=postgres://crm_user:crm_pass@localhost:5432/crm_db
```

Установите драйвер: `pip install "psycopg[binary]"` и заново выполните `migrate` + `seed_demo`.

## Сценарий демонстрации (для защиты)

1. Войти под `admin` → панель управления показывает счётчики и мини-воронку.
2. **Клиенты** → найти клиента, открыть карточку (видны контакты, список сделок, история).
3. **Сделки → Воронка** → перетащить карточку между колонками — статус обновится автоматически.
4. На карточке клиента добавить взаимодействие (звонок/встречу) — появится в ленте.
5. **Отчёты** → графики Chart.js, фильтр по периоду, экспорт CSV.
6. Открыть **Пользователи** → создать нового, или войти под `viewer` — увидеть, что кнопки редактирования скрыты.
7. Открыть **Этапы воронки** → добавить/изменить этап, перейти на доску — изменения отражены.

## Безопасность и нефункциональные требования

- CSRF, `LoginRequiredMixin`, ролевой доступ через декоратор `accounts.permissions.role_required`
- Пароли — `PBKDF2` (Django default)
- Все queryset'ы используют `select_related` / `prefetch_related` (требование производительности ≤ 2с)
- `SECRET_KEY` и `DEBUG` вынесены в `.env`
- Для бэкапа: `python manage.py dumpdata --indent 2 > backup.json`
